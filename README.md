# QQSpaceAutoLike

`QQSpaceAutoLike` 是一个仅适用于手机 QQ 的 Android 原型项目。

当前版本同时保留两种方法，并在 App 里分成两个独立页面：
- `旧方法`：无障碍进入好友动态页面后，直接识别卡片并尝试点按点赞按钮
- `新方法`：后台定时拉取 QQ 空间好友动态，先放进本地待补赞队列，只有当你这台手机上的 `QQ` 进入前台时，才按随机延迟执行一轮补赞

它不是“好友一发就秒赞”的工具，也不是 QQ 插件注入方案。

## 适用范围

- 仅适用于 `手机 QQ`
- 两种方法都依赖 `AccessibilityService`
- 新方法额外依赖你手动提供 `Qzone Cookie`

不适用：
- TIM
- QQ HD / 平板变体
- PC QQ
- 需要 Root / Xposed / LSPosed 注入的插件场景

## 两种方法

```text
旧方法
  手机 QQ 前台
    ↓
  进入好友动态
    ↓
  识别卡片与点赞按钮
    ↓
  直接点按

新方法
后台轮询 Qzone feed
  ↓
发现新的好友说说链接
  ↓
跳过广告 / 跳过自己的说说 / 本地去重
  ↓
写入待补赞队列
  ↓
等待你打开手机 QQ
  ↓
按延迟窗口 + 随机等待执行一轮补赞
```

## 为什么保留两种方法

旧方法的优点：
- 不需要 Cookie
- 只靠手机本机就能跑

旧方法的问题：
- 长短卡片混排时容易误判
- 有时会点进图片
- 广告角标滑出屏幕后容易误赞
- 很难做成“不是秒赞、但又接近真人”的行为

新方法的优点：
- 不依赖屏幕卡片识别作为主路径
- 更适合做“延迟补赞”而不是“秒赞”
- 只有你打开手机 QQ 时才会放行点赞

新方法的问题：
- 需要自己提供 `Qzone Cookie`
- Cookie 有失效风险
- 非官方网页协议可能变化

所以现在不是删掉旧方法，而是让用户自己选：
- 要简单直接就用旧方法
- 要更像“延迟补赞”就用新方法

## 新方法行为

- 后台每隔一段时间轮询一次好友动态
- 新动态不会立刻点赞
- 每条动态会先等待你设置的最短延迟窗口
- 只有你打开手机 QQ 时，才会在本次前台会话里随机补赞若干条

可调项：
- 轮询间隔
- 最短延迟补赞时间
- 每次打开 QQ 最多补赞多少条
- 会话最长持续时间
- 是否跳过广告
- 是否启用随机等待

## 新方法需要你提供的内容

应用内至少需要：
- `QQ 号`
- `Qzone Cookie`

Cookie 至少应包含：
- `p_skey`
或
- `skey`

没有这些字段时，无法计算 `g_tk`，也就无法请求 Qzone 接口。

### Cookie 获取方式

最简单的是用电脑浏览器：

1. 登录 `https://user.qzone.qq.com/你的QQ号`
2. 按 `F12` 打开开发者工具
3. 切到 `Network / 网络`
4. 刷新页面
5. 找到 `feeds_html_act_all` 或任意 `user.qzone.qq.com/proxy/domain/...` 请求
6. 在 `Headers / 请求头` 里复制 `cookie:` 整行
7. 粘贴到应用里的新方法页面

### Cookie 会不会一直有效

不会。

常见失效原因：
- QQ 重新登录
- 网页登录态自然过期
- 长时间未使用
- 触发风控或验证
- 修改密码或安全设置

所以更准确的说法是：
- `Cookie` 往往能连续用一段时间
- 但不能假设它永久有效
- 失效后重新抓一份最新的通常就行

## 不是插件注入

这个仓库当前不是 QQ 插件，也不修改 QQ 安装包。

原因很简单：
- 插件注入通常意味着进入 QQ 进程做 hook
- 风险和维护成本都显著更高
- QQ 版本一变，hook 点往往就要跟着改

所以这里优先保留的是：
- 独立 Android 应用
- 无障碍监听前台状态
- 可选的 Qzone Cookie 协议补赞

## Build

要求：
- `JDK 17`
- Android SDK

本地构建：

```bash
./gradlew assembleDebug
```

## 安装与测试

1. 安装 APK
2. 打开应用
3. 启用无障碍服务
4. 在应用里选择 `旧方法` 或 `新方法`
5. 如果选新方法，填入 `QQ号` 和 `Qzone Cookie`
6. 设置对应方法的参数
7. 打开手机 QQ，观察是否开始一轮补赞

## 风险说明

- `Cookie` 本质上是登录态，泄露后风险很高
- 非官方网页协议可能随时变化
- Qzone 侧可能出现风控、验证、登录失效
- 即使不是秒赞，也不能保证完全拟人

## 现状

当前版本状态：
- `旧方法` 已恢复为独立可选页
- `新方法` 已改成 `A1` 会话触发模式
- 两种方法不再混在同一页里
- 新方法页内置了 Cookie 获取教程和失效说明
